# 전시장 개관 인트로 (Gallery Opening Intro) — 설계 문서

작성일: 2026-07-08
대상 파일: `index.html` (단일 파일 SPA, GitHub Pages)

## 목표

청첩장 진입 순간 하객이 "우와" 하는 오프닝을 넣는다. 기존 "THE EXHIBITION
OF TWO"(미술관/전시회) 컨셉을 그대로 살려, **어두운 전시장에 조명이 켜지며
개관하는** 연출로 커버 화면을 드러낸다. 소리는 넣지 않는 대신 클라이맥스에
**화면 흔들림(카메라 셰이크)**으로 임팩트를 준다.

## 사용자 경험 (연출 흐름)

전체 ~2.4초. 언제든 **탭 또는 스크롤로 즉시 건너뛰기** 가능.

| 시점(대략) | 장면 |
|-----------|------|
| 0.0s | 화면 전체를 어두운 전시장(따뜻한 차콜 `#17140f`)이 덮음. 스크롤 잠김. 은은한 비네팅 + 기존 벽 그레인 질감 |
| 0.3s | 중앙 상단에서 금빛 스포트라이트가 서서히 번지며 밝아짐 ("조명이 켜진다") |
| 0.8s | 스포트라이트 안에서 `THE EXHIBITION OF TWO`가 한 글자씩 금색으로 각인, 아래 금색 헤어라인이 좌→우로 그어짐 |
| 1.4s | `이종호 & 김희정` 이름이 아래에 페이드인 |
| 1.9s | 어둠이 부드럽게 걷히며(페이드 + 살짝 확대) 실제 커버가 드러남. **이 순간 화면이 쿵 하고 흔들림** + 모바일 진동(best-effort) |
| ~2.4s | 커버의 기존 요소들이 순차적으로 떠오름(기존 staged reveal) → 본편으로 자연스럽게 연결 |

## 동작 원칙 / 요구사항

- **재생 시점**: 방문할 때마다 재생. 상태 저장(localStorage 등) 없음.
- **건너뛰기**: 오버레이 표시 중 화면 탭 또는 스크롤 입력 시 즉시 종료(오버레이 제거 + 스크롤 잠금 해제 + 커버 리빌 시작).
- **화면 흔들림**: "개관" 순간 커버 콘텐츠 래퍼에 짧은(~0.4s) 셰이크. 미세한 translate(±3px 내외) + 소량 rotate(±0.4deg)로 카메라가 흔들리는 느낌. 사운드 대체 임팩트.
- **진동(햅틱)**: 셰이크 시점에 `navigator.vibrate?.(...)` best-effort 호출. 차단되면 무해하게 무시(오디오 없이 촉각만).
- **접근성**: `prefers-reduced-motion: reduce`인 기기는 인트로/셰이크 생략하고 즉시 커버 표시.
- **FOUC 방지**: 첫 페인트부터 오버레이가 화면을 덮어, 커버가 먼저 번쩍 보였다가 가려지는 일이 없어야 함. 오버레이는 초기 마크업에 포함되고 스크립트 실행 전에도 화면을 덮는다.
- **타이밍 튜닝**: 각 비트의 지연/길이를 한 곳(상단의 명명된 상수/CSS 변수)에서 조절 가능하게 둔다. 최종 길이는 미리보기 후 사용자가 확정.

## 기술 설계

새 파일·라이브러리 없음. `index.html` 안에서 해결.

### 1) 마크업 — 오버레이

`root`(`ref=setRoot`) 하위, 콘텐츠보다 앞에 오버레이 하나 추가:

- `<div data-opening>`: `position:fixed; inset:0; z-index:9999`(라이트박스 등 기존 z-index보다 크게). 배경 `#17140f`.
  내부에 스포트라이트 레이어(라디얼 그라디언트), 타이틀 텍스트, 이름, 헤어라인.
- 오버레이는 초기 마크업에 존재하고 CSS `@keyframes`로 **스크립트 없이도** 즉시 애니메이션이 시작된다(FOUC 방지 + JS 실패 시에도 최소 동작).
- 오버레이는 `position:fixed`이며 **셰이크되는 콘텐츠 래퍼의 형제**로 둔다 → 셰이크의 transform이 fixed 오버레이의 컨테이닝 블록에 영향 주지 않게 분리.

### 2) 스타일 — `<style>`의 `@keyframes`

- `openingSpotlight`: 스포트라이트 밝기/크기 상승.
- 타이틀 글자별 각인: 글자를 `<span>`으로 쪼개 순차 `opacity`/`translateY` (또는 단어 단위 stagger). CSS 지연으로 구현.
- `openingLine`: 헤어라인 `scaleX(0→1)`.
- `openingLift`: 오버레이 페이드아웃 + `scale(1→1.06)` 로 "걷히는" 마무리.
- `openingShake`: 콘텐츠 래퍼 흔들림(짧은 다단계 translate/rotate, ease-out).
- `@media (prefers-reduced-motion: reduce)`: 오버레이 즉시 숨김, 셰이크 없음.

### 3) 로직 — `setupOpening(root)` (componentDidMount에 추가)

책임(단일 목적: 인트로 수명 관리):
- reduced-motion이면 오버레이 즉시 제거 후 커버 리빌 시작하고 종료.
- 그렇지 않으면:
  - 개관 시점(~1.9s)에 콘텐츠 래퍼에 셰이크 클래스 부여 + `navigator.vibrate?.([20,40,20])` best-effort.
  - 종료 시점(~2.4s)에 오버레이 제거, `body`/스크롤 잠금 해제.
  - 탭/스크롤 리스너로 조기 종료(위 종료 로직을 공유하는 `finish()` 한 번만 실행되게 가드).
- `componentWillUnmount`에서 리스너/타이머 정리(기존 패턴과 동일: `this._timers`, 명시적 removeEventListener).

### 커버 리빌과의 연결

현재 `setupReveal`은 요소를 `opacity:0`으로 두고 뷰포트 진입 시 드러내며,
로드 직후(120/350/800/1500ms) 재검사한다. 오버레이가 덮인 동안 커버가 뒤에서
먼저 드러나면 "개관 후 작품이 하나씩" 연출이 낭비된다.

해결: **커버 섹션의 리빌을 인트로 종료 이후로 지연**한다. `setupOpening`이
`finish()` 시점에 리빌 재검사를 트리거하고, 인트로가 끝나기 전에는 커버가
드러나지 않도록 초기 검사 타이밍을 인트로 종료 기준으로 맞춘다. (본문 하단
섹션들은 원래대로 스크롤 시 드러남 — 영향 없음.)

## 스크롤 잠금

인트로 동안 `document.body`(또는 스크롤 컨테이너) `overflow:hidden`으로 잠그고,
`finish()`에서 해제. 잠금 실패 시에도 오버레이가 시각적으로 덮으므로 치명적이지 않음.

## 엣지 케이스 / 에러 처리

- JS 미실행/오류: CSS만으로 인트로가 재생되고 마지막 프레임에서 오버레이가
  투명해지도록(`animation-fill-mode: forwards` + 최종 `pointer-events:none`) 하여
  페이지가 잠기지 않게 한다. (JS는 정리·스킵·셰이크 담당, 표시 자체는 CSS가 보장.)
- `navigator.vibrate` 미지원/차단: optional chaining으로 무해하게 통과.
- 매우 느린 기기: 타이밍이 늘어져도 탭/스크롤 스킵으로 즉시 벗어날 수 있음.

## 테스트 / 검증

- 데스크톱·모바일 폭에서 실제 로드하여 시퀀스 육안 확인(길이 튜닝 포함).
- 탭 스킵 / 스크롤 스킵 각각 즉시 종료되는지.
- OS "동작 줄이기" 켠 상태에서 인트로·셰이크 생략되고 커버 즉시 표시되는지.
- 인트로 종료 후 커버 요소들이 순차 리빌되는지, 이후 스크롤 리빌 정상인지.
- (검증 코드) 셰이크/종료가 **한 번만** 실행되는지 `finish()` 가드에 대한 최소 자체 점검.
